# SIM MANAGEMENT SAAS - PROJECT CONTEXT

## PROJECT OVERVIEW
```
Project: SIM Management SaaS
Type: Multi-tenant SaaS
Version: 1.0.0
Status: Development
```

## SAAS WORKFLOW
1. Admin creates client account
2. Client logs into web dashboard
3. Client adds SIM cards and recharge details
4. System stores data securely in cloud database
5. Mobile app (if enabled) syncs call logs
6. Dashboard displays reports, alerts, and analytics
7. Subscription plan controls features and limits

## TECH STACK

### Backend
- Runtime: Node.js 18+
- Framework: Express.js
- Database: MongoDB
- Authentication: JWT
- Validation: Joi / Express Validator
- File Processing: Multer, XLSX

### Frontend
- Framework: React 18
- Styling: Tailwind CSS
- State: Context API / Redux
- Routing: React Router v6
- HTTP: Axios
- Charts: Chart.js / Recharts

### Infrastructure
- Deployment: Docker + Kubernetes ready
- CI/CD: GitHub Actions
- Monitoring: Winston + Morgan

## DATABASE SCHEMA

### Users Collection
```javascript
{
  _id: ObjectId,
  email: String (unique, indexed),
  password: String (hashed),
  name: String,
  role: String (super_admin|admin|user),
  companyId: ObjectId (ref: companies, null for super_admin),
  phone: String,
  isActive: Boolean,
  lastLogin: Date,
  createdAt: Date,
  updatedAt: Date
}
```

### Companies Collection
```javascript
{
  _id: ObjectId,
  name: String,
  email: String (unique, indexed),
  phone: String,
  address: String,
  subscriptionId: ObjectId (ref: subscriptions),
  subscriptionStartDate: Date,
  subscriptionEndDate: Date,
  isActive: Boolean,
  settings: Object,
  createdAt: Date,
  updatedAt: Date
}
```

### SIMs Collection
```javascript
{
  _id: ObjectId,
  companyId: ObjectId (ref: companies, indexed),
  simNumber: String (unique, indexed),
  operator: String,
  mobileNumber: String,
  assignedTo: ObjectId (ref: users),
  status: String (active|inactive),
  activationDate: Date,
  notes: String,
  createdAt: Date,
  updatedAt: Date
}
```

### Recharges Collection
```javascript
{
  _id: ObjectId,
  companyId: ObjectId (ref: companies, indexed),
  simId: ObjectId (ref: sims),
  amount: Number,
  rechargeDate: Date,
  nextRechargeDate: Date,
  plan: String,
  validity: Number,
  status: String (pending|completed|failed),
  createdAt: Date,
  updatedAt: Date
}
```

### CallLogs Collection
```javascript
{
  _id: ObjectId,
  companyId: ObjectId (ref: companies, indexed),
  simId: ObjectId (ref: sims),
  phoneNumber: String,
  callType: String (incoming|outgoing|missed),
  duration: Number,
  timestamp: Date,
  syncedFrom: String (web|mobile),
  createdAt: Date
}
```

### Notifications Collection
```javascript
{
  _id: ObjectId,
  companyId: ObjectId (ref: companies, indexed),
  userId: ObjectId (ref: users),
  type: String (recharge_due|inactive_sim|system),

  message: String,
  isRead: Boolean,
  sentVia: [String] (email|sms|in_app),

```

### Subscriptions Collection
```javascript
{
```

## BUSINESS RULES

### Role Hierarchy
1. Super Admin
   - Platform owner
   - Creates companies
   - Creates Admins
   - Assigns subscription plans
   - Views all data
   - System configuration

2. Admin (Company Level)
   - Manages company SIMs
   - Manages company recharges
   - Manages company users
   - Views company reports
   - Configures company settings

3. User
   - Views assigned SIMs
   - Updates SIM status
   - Syncs call logs (mobile)

### Data Isolation
- All company data filtered by companyId
- Super Admin bypasses company filter
- Indexed queries for performance

### Subscription Limits
- Max SIMs per plan
- Max users per plan
- Feature toggles per plan
- Grace period for expired plans

### Reminder System
- Recharge reminders: 3 days before expiry
- Inactive SIM alerts: 7 days inactive
- Daily cron job execution
- Notification history tracking
